Existe uma diferença grande entre discutir código com uma IA e ter o código de verdade editado por ela — com a sua aprovação a cada passo. O Claude Code é exatamente essa segunda categoria: um agente que lê, entende e modifica arquivos reais do seu projeto. O que é o Claude Code

Claude Code é um agente de programação da Anthropic que entende o contexto completo do seu projeto — não só um arquivo isolado — e consegue ler, propor e aplicar mudanças reais de código, executar comandos e usar ferramentas como Git, sempre com sua aprovação no caminho.

A diferença central em relação a colar código numa conversa comum é a ação direta: em vez de receber uma sugestão para você copiar e aplicar manualmente, o Claude Code edita os arquivos do seu disco, dentro dos limites que você autorizar.

Os formatos disponíveis — qual escolher

⌨️
Terminal (CLI)
A forma original, direta na linha de comando. Ideal para quem já trabalha assim no dia a dia.
🖥️
App Desktop
Interface visual com sessões em paralelo, terminal integrado e comparação visual de mudanças. Disponível para Mac e Windows.
🧩

Extensão que roda dentro do próprio editor, com feedback visual e diffs embutidos.
🌐

Você pode combinar formatos livremente — começar uma sessão no terminal e continuar no app Desktop, ou o caminho inverso, sem perder o contexto.

Como instalar, passo a passo

1

Confirme seu plano

Você precisa de uma conta paga do Claude (Pro, Max, Team ou Enterprise) ou de uma conta de Console com créditos. O plano gratuito não dá acesso ao Claude Code.

2

Use o instalador nativo

O método recomendado atualmente é o instalador nativo do seu sistema — mais rápido e com atualização automática. O método antigo via npm foi descontinuado pela Anthropic.

3

Abra um novo terminal

Feche e reabra a janela do terminal para garantir que o sistema reconheça o novo caminho de instalação.

4

Inicie o Claude Code

Navegue até a pasta do seu projeto e digite o comando para iniciar uma sessão.

5

Autentique sua conta

Na primeira vez, uma janela do navegador abre automaticamente para login. Depois disso, a sessão fica salva.

Importante: sempre inicie o Claude Code a partir da pasta raiz do seu projeto. É assim que ele entende a estrutura completa do código — sem isso, as respostas tendem a ficar genéricas.

Sua primeira sessão

Comece com um projeto que você já conhece bem, e peça algo fácil de verificar — "o que essa função retorna?" ou "onde essa variável é usada?". Isso ajuda a calibrar a confiança antes de pedir mudanças mais importantes.

A interação é uma conversa, não um pedido único: o Claude lê os arquivos relevantes, propõe uma mudança, e você aprova, ajusta ou recusa antes de qualquer coisa ser escrita no disco.

Dica de especificidade

Pedidos vagos como "melhore esse módulo" tendem a gerar mudanças amplas demais. Pedidos específicos como "troque o nome da função X para Y e atualize quem chama ela" reduzem efeitos colaterais indesejados.

Como funciona o controle sobre as mudanças

Por padrão, o Claude Code pede aprovação antes de cada escrita de arquivo ou execução de comando — nada muda sem sua confirmação explícita. Esse comportamento pode ser ajustado conforme você ganha confiança, liberando algumas ações específicas sem precisar aprovar uma por uma.

Mesmo com mudanças já aplicadas, ferramentas de controle de versão como o Git permitem ver exatamente o que foi alterado e revertam arquivos individuais sem perder o restante do trabalho — uma rede de segurança adicional além da aprovação prévia.

O arquivo CLAUDE.md — contexto do projeto

Um arquivo chamado CLAUDE.md, colocado na raiz do projeto, funciona como uma "ficha de instruções" que o Claude Code lê automaticamente em cada sessão — convenções de código, preferências de estilo, contexto sobre a arquitetura, o que evitar.

Isso evita repetir as mesmas instruções em toda conversa nova. Se o Claude parecer estar ignorando algo que está nesse arquivo, geralmente é porque ele já tinha lido a versão anterior antes da sua edição — nesse caso, reiniciar a sessão resolve.

Erros comuns de quem está começando

1
Instalar pelo método antigo (npm). Esse caminho foi descontinuado — use o instalador nativo recomendado para evitar problemas de atualização.
2
Iniciar a sessão fora da pasta do projeto. Sem o contexto da pasta raiz, as respostas ficam genéricas e menos úteis.
3
Pedidos vagos demais. "Melhore o código" gera mudanças imprevisíveis. Pedidos específicos, com nome de arquivo e comportamento esperado, funcionam muito melhor.
4
Tentar usar com plano gratuito. Confirme seu plano antes de instalar — economiza tempo de frustração.
5
Não revisar antes de aprovar. O sistema de aprovação existe por um motivo — usar ele com atenção é o que mantém o controle nas suas mãos.
