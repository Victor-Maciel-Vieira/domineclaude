Claude Code: as novidades de julho 2026 (/checkup, navegador embutido e mais)

📅 Julho 2026 ⏱ 6 min de leitura ✍️ Kortex Evolution

O Claude Code recebeu uma leva pesada de atualizações nas últimas semanas de julho de 2026 — e a mais notável é a transformação do comando /doctor em uma ferramenta completa de manutenção do seu ambiente de desenvolvimento. Junto vieram navegador embutido no desktop, proteções no modo automático e melhorias na visualização de agentes.

Em resumo: o comando /doctor (agora também chamado /checkup) deixou de ser apenas um relatório e passou a diagnosticar e corrigir problemas automaticamente. O Claude Code no desktop ganhou navegador embutido, e o modo automático ficou mais seguro contra alterações acidentais.

/doctor virou /checkup: manutenção automática do seu setup

Antes, o /doctor era um verificador de saúde somente leitura: mostrava o status da instalação, configurações e uso de contexto, com um botão para aplicar correções simples. A partir da versão 2.1.205 (8 de julho de 2026), ele virou uma checagem completa que diagnostica e já propõe correções — e /checkup é seu apelido, os dois comandos fazem exatamente a mesma coisa.

O que o /checkup verifica e pode corrigir:

  • Skills, MCPs e plugins não utilizados versus o custo de contexto que eles consomem.
  • Arquivos CLAUDE.md duplicados — deduplica o seu CLAUDE.md local contra o que já está versionado no repositório.
  • CLAUDE.md muito grande — sugere dividir um arquivo raiz inchado em arquivos aninhados e skills.
  • Hooks lentos que estão deixando sua sessão mais devagar.
  • Versão desatualizada do Claude Code.
  • Modo automático desativado — sugere ativar se fizer sentido para seu fluxo.
  • Comandos somente leitura que você vem negando repetidamente — sugere pré-aprovação.

O comando reporta os achados primeiro e pede confirmação antes de mudar qualquer coisa. Você escolhe: limpar tudo, escolher categorias específicas, ou não mudar nada. Mudanças em CLAUDE.md vão para sua árvore de trabalho para revisão normal via git diff antes do commit.

Navegador embutido no Claude Code Desktop

O Claude Code no desktop agora tem um navegador embutido. O Claude pode abrir documentação, designs ou qualquer site, e ler, clicar e interagir com páginas do mesmo jeito que já fazia com prévia de servidores de desenvolvimento local. O navegador é isolado (sandboxed) e configurável: você escolhe se as sessões de navegação persistem, e classificadores de segurança revisam ações em sites externos.

Isso é particularmente útil para quem trabalha consultando documentação de API constantemente, ou revisando designs no Figma enquanto codifica — tudo dentro do mesmo ambiente, sem trocar de janela.

Modo automático mais seguro

O modo automático (auto mode) do Claude Code ganhou várias camadas de proteção:

  • Bloqueio de alteração em transcripts: o modo automático agora impede que a sessão mexa nos próprios arquivos de transcript, evitando que aprovações fabricadas dentro do histórico sejam usadas indevidamente.
  • Confirmação antes de rm -rf em variável não resolvida: se o Claude não consegue resolver uma variável a partir do contexto, ele pergunta antes de rodar um comando destrutivo.
  • Sub-agentes agora são avaliados pelo classificador antes de serem lançados, fechando uma brecha onde um sub-agente poderia pedir uma ação bloqueada sem revisão.
  • Ativado por padrão em Bedrock, Vertex AI e Foundry, sem precisar de opt-in via variável de ambiente (pode ser desativado com disableAutoMode).

Melhorias na visualização de agentes

O comando claude agents ficou mais claro: as linhas agora mostram uma palavra de estado colorida e um título escrito por classificador, em vez de texto bruto de chamada de ferramenta. Sessões que editam, mesclam, comentam ou enviam para um Pull Request existente agora aparecem vinculadas na lista de agentes.

Outras correções relevantes

  • Downloads do auto-updater agora fazem streaming direto para o disco, reduzindo em cerca de 400 MB o uso de memória de pico do atualizador.
  • Notificações de tarefas em segundo plano agora declaram explicitamente quando nenhuma entrada humana ocorreu, prevenindo que aprovações fabricadas dentro do transcript sejam executadas.
  • Melhor qualidade nos achados do /code-review rodando no Opus 4.8, em todos os níveis de esforço.
  • Corrigido travamento do terminal ao rodar respostas com listas, tabelas ou blocos de código muito longos.

Como aproveitar essas atualizações hoje

Se você já usa o Claude Code, rode claude update para garantir a versão mais recente, depois execute /checkup (ou /doctor, é o mesmo comando) para fazer uma limpeza completa do seu ambiente. É recomendável rodar esse comando periodicamente — especialmente após atualizações grandes ou quando o Claude Code parecer mais lento que o normal.
